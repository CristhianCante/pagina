Ráster en PostGIS

Almacenando y analizando datos matriciales directamente en la base de datos.

1. ¿Qué son los Datos Ráster?

Además de los datos vectoriales (puntos, líneas, polígonos), PostGIS también soporta datos ráster. Un ráster es un modelo de datos que representa el espacio como una matriz o cuadrícula de celdas (píxeles). Cada píxel contiene un valor que representa una característica de esa área, como la elevación, la temperatura, o un color en una imagen.

Son ideales para representar fenómenos continuos como fotografías aéreas, imágenes de satélite o modelos digitales de elevación.

1.1. Ráster como Geometrías

PostGIS permite tratar los datos ráster de forma similar a las geometrías, ofreciendo funciones para extraer su representación vectorial.

  • ST_Envelope(raster): Devuelve el rectángulo mínimo que encierra completamente el ráster.
  • ST_ConvexHull(raster): Crea el menor polígono convexo que contiene todos los píxeles con datos.
  • ST_Polygon(raster): Convierte los píxeles con datos en una geometría de polígono, delineando sus bordes.

2. Habilitar y Usar Ráster en PostGIS

Para trabajar con datos ráster, primero debemos asegurarnos de que la extensión postgis_raster esté habilitada en nuestra base de datos. 

CREATE EXTENSION postgis_raster;

2.1. Crear un Ráster desde una Geometría

Podemos generar un ráster a partir de una geometría vectorial con la función ST_AsRaster. Esto es útil para "rasterizar" vectores.

-- Crear una tabla de rásters y generar un ráster a partir del texto "Hello"
CREATE TABLE rasters (name text, rast raster);

INSERT INTO rasters (name, rast)
SELECT 
    f.word, 
    ST_AsRaster(
        ST_Letters(f.word), -- Genera la geometría del texto
        150, 150,           -- Ancho y alto en píxeles
        '8BUI'              -- Tipo de dato de píxel
    )
FROM (VALUES ('Hello'), ('Raster')) AS f(word);

Posteriormente podremos ver el raster que creamos, a partir de una consulta:

-- Visualización del raster a través de la consulta
SELECT 'data:image/png;base64,'
    encode(ST_AsPNG(rast),'base64')
    FROM rasters
    WHERE name = 'Hello';

3. Cargar Datos Ráster con `raster2pgsql`

La forma más común de cargar archivos ráster (como GeoTIFFs) en PostGIS es usando la herramienta de línea de comandos raster2pgsql. Esta herramienta convierte el archivo ráster en sentencias SQL que crean y pueblan una tabla en la base de datos.

3.1. Uso Básico

Para usarla, abre la terminal o símbolo del sistema, navega a la carpeta bin de tu instalación de PostgreSQL y ejecuta el comando.

La sintaxis general es:

raster2pgsql [opciones] /ruta/al/raster.tif nombre_esquema.nombre_tabla | psql -d nombre_db -U usuario

Algunas opciones importantes son:

  • -s <srid>: Asigna un Sistema de Referencia Espacial.
  • -I: Crea un índice espacial GiST sobre la columna ráster.
  • -C: Aplica restricciones de ráster para asegurar la integridad.
  • -M: Calcula estadísticas para optimizar consultas.
  • -t <tamaño>: Divide el ráster en teselas (tiles), ej: -t 100x100. Es muy recomendado para rásters grandes.

3.2. Ejemplo de Carga

Vamos a cargar una imagen satelital de Chía.

raster2pgsql.exe -s 32618 -I -C -M C:\Rasters\chia.tif public.chia | psql -d postgis_bogota -U postgres

Este comando cargará el archivo chia.tif en una tabla llamada chia dentro del esquema public de la base de datos postgis_bogota.

4. Análisis Ráster: Cálculo de NDVI

Una vez que tenemos los datos ráster en la base de datos, podemos realizar análisis directamente con SQL. Un ejemplo clásico es el cálculo del Índice de Vegetación de Diferencia Normalizada (NDVI), que se usa para evaluar la salud de la vegetación.

La fórmula es: NDVI = (NIR - RED) / (NIR + RED), donde NIR es la banda del Infrarrojo Cercano y RED es la banda del Rojo.

Suponiendo que en nuestra imagen de Chía, la banda 4 es NIR y la banda 3 es RED, podemos usar la función ST_MapAlgebra para aplicar esta fórmula a cada píxel.

-- Crear una nueva tabla 'ndvi' con el resultado del cálculo
CREATE TABLE ndvi AS
SELECT 
    ST_MapAlgebra(
        c1.rast, 1, -- Banda NIR de la primera fuente (alias rast1)
        c2.rast, 1, -- Banda RED de la segunda fuente (alias rast2)
        '(([rast1.val] - [rast2.val])::float8 / ([rast1.val] + [rast2.val])::float8)', -- Expresión
        '32BF' -- Tipo de píxel para el resultado
    ) AS rast
FROM 
    (SELECT ST_Band(rast, 4) AS rast FROM chia) AS c1,
    (SELECT ST_Band(rast, 3) AS rast FROM chia) AS c2;

El resultado es una nueva tabla ráster llamada ndvi, que podemos visualizar en QGIS para ver las zonas con mayor o menor vigor vegetal.
