Relaciones Espaciales

Cómo describir e identificar los vínculos que existen entre diferentes geometrías.

1. ¿Qué son las Relaciones Espaciales?

Las relaciones espaciales son vínculos que describen cómo interactúan dos o más geometrías en el espacio. Responden a preguntas como: ¿Está este punto dentro de este polígono? ¿Se cruza esta calle con este río? ¿Qué barrios colindan con este otro?

En PostGIS, estas relaciones se evalúan mediante funciones que, en su mayoría, devuelven un valor booleano (true o false) dependiendo de si la condición espacial se cumple o no.

2. Funciones de Relaciones Espaciales

Para los siguientes ejemplos, utilizaremos datos de la localidad de Suba en Bogotá.

Las funciones se pueden agrupar en tres categorías principales:

2.1. Relaciones de Contención

Evalúan si una geometría está contenida o es igual a otra.

ST_Contains(geomA, geomB)

Devuelve true si la geometría A contiene completamente a la geometría B. Ninguna parte de B puede estar fuera de A.

Ejemplo: ¿Cuáles incidentes ocurrieron dentro del barrio "EL PLAN"?

SELECT i.numero_in, i.servicio, b.nombre AS barrio
FROM incidentes i, barrios b
WHERE ST_Contains(b.geom, i.geom)
  AND b.nombre = 'EL PLAN';

ST_Within(geomA, geomB)

Es la función opuesta a ST_Contains. Devuelve true si la geometría A está completamente contenida dentro de la geometría B.

Ejemplo: ¿Cuáles calles están completamente dentro del barrio "TUNA RURAL"?

SELECT m.mvietiquet, m.mvinantigu, b.nombre AS barrio
FROM malla_vial m, barrios b
WHERE ST_Within(m.geom, b.geom)
  AND b.nombre = 'TUNA RURAL';

ST_Touches(geomA, geomB)

Devuelve true si las geometrías se tocan en sus bordes, pero sus interiores no se intersectan.

Ejemplo: ¿Qué barrios colindan (tocan) con el barrio "COSTA AZUL"?

SELECT b1.nombre
FROM barrios AS b1, barrios AS b2
WHERE ST_Touches(b1.geom, b2.geom)
  AND b2.nombre = 'COSTA AZUL';

ST_Equals(geomA, geomB)

Devuelve true si ambas geometrías son espacialmente idénticas (tienen los mismos puntos y forma).

Ejemplo: ¿Cuáles incidentes ocurrieron exactamente en el mismo lugar (aunque sean registros diferentes)?

SELECT e.id, e.servicio
FROM incidentes AS e, incidentes AS c
WHERE ST_Equals(e.geom, c.geom)
  AND e.id <> c.id;

2.2. Relaciones de Intersección

Evalúan si las geometrías comparten alguna porción de espacio.

ST_Intersects(geomA, geomB)

Es la función más utilizada. Devuelve true si las geometrías comparten cualquier porción de espacio, ya sea un punto, una línea o un área.

Ejemplo: ¿Cuáles vías se intersectan con el barrio "SAN JOSE DE BAVARIA"?

SELECT m.mv_nombre, m.id AS id_vias
FROM barrios b, malla_vial m
WHERE ST_Intersects(b.geom, m.geom)
  AND b.nombre = 'SAN JOSE DE BAVARIA';

ST_Disjoint(geomA, geomB)

Es lo opuesto a ST_Intersects. Devuelve true si las geometrías no comparten ningún espacio en común (están separadas).

Ejemplo: ¿Qué barrios no tienen ninguna colindancia con el barrio "EL PINO"?

SELECT no_colindantes.nombre
FROM barrios AS el_pino, barrios AS no_colindantes
WHERE ST_Disjoint(el_pino.geom, no_colindantes.geom)
  AND el_pino.nombre = 'EL PINO';

ST_Crosses(geomA, geomB)

Devuelve true si las geometrías se cruzan y la dimensión de la intersección es menor que la dimensión de al menos una de las geometrías. Típicamente se usa para ver si una línea cruza un polígono o si dos líneas se cruzan en un punto.

Ejemplo: ¿Qué vías atraviesan el barrio "CIUDAD JARDIN NORTE"?

SELECT m.mv_nombre
FROM barrios b, malla_vial m
WHERE b.nombre = 'CIUDAD JARDIN NORTE'
  AND ST_Crosses(b.geom, m.geom);

ST_Overlaps(geomA, geomB)

Devuelve true si las geometrías se intersectan, tienen la misma dimensión, pero no son completamente iguales.

Ejemplo: ¿Qué barrios se superponen con la localidad de SUBA (útil para encontrar barrios en el borde)?

SELECT b.nombre, l.locnombre
FROM barrios b, localidad l
WHERE ST_Overlaps(l.geom, b.geom)
  AND l.locnombre = 'SUBA';

2.3. Relaciones de Distancia

ST_Distance(geomA, geomB)

Esta función es un poco diferente, ya que no devuelve un booleano, sino un número (double precision) que representa la distancia más corta entre dos geometrías, en las unidades de su SRID.

Ejemplo: ¿Cuál es el incidente más cercano a la Avenida Carrera 104?

SELECT i.servicio, m.mvietiquet, i.geom
FROM incidentes i, malla_vial m
WHERE m.mvietiquet = 'Avenida Carrera 104'
ORDER BY ST_Distance(m.geom, i.geom) ASC
LIMIT 1;

ST_DWithin(geomA, geomB, distancia)

Esta función sí es un predicado espacial. Devuelve true si las geometrías están a una distancia específica la una de la otra. Es mucho más eficiente que usar ST_Distance en una cláusula WHERE porque puede usar índices espaciales.

Ejemplo: ¿Cuáles son los incidentes que ocurrieron a 100 metros o menos del barrio "SUBA URBANO"? (Nota: La distancia debe estar en las unidades del SRID).

SELECT i.id, i.servicio
FROM incidentes i, barrios b
WHERE b.nombre = 'SUBA URBANO'
  AND ST_DWithin(b.geom, i.geom, 100);
