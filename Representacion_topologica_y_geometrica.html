Representación Topológica y Geométrica

Comprendiendo la estructura interna de las TopoGeometrías, Capas y Relaciones.

1. De Geometría a TopoGeometría

Cuando trabajamos con un modelo topológico, no almacenamos geometrías simples (geometry), sino TopoGeometrías (topogeometry). Una TopoGeometría no es una forma autocontenida, sino una referencia a un conjunto de primitivas topológicas (nodos, aristas, caras) que, juntas, componen la forma deseada.

Esta estructura es la que permite que dos polígonos adyacentes compartan la misma arista, garantizando que no haya huecos ni superposiciones entre ellos.

1.1. Capas, TopoGeometrías y TopoElementos

La organización interna de una topología en PostGIS se basa en tres conceptos:

  • Primitivas: Son los elementos básicos e indivisibles: Nodos, Aristas y Caras.
  • TopoElemento: Es una referencia que puede apuntar a una primitiva (ej. la arista con ID 5) o a otra TopoGeometría de una capa inferior.
  • TopoGeometría: Es un conjunto de uno o más TopoElementos.
  • Capa (Layer): Es un contenedor que agrupa TopoGeometrías de un mismo tipo (puntos, líneas o polígonos). Cada columna de tipo `topogeometry` en una tabla está asociada a una capa dentro de la topología.

2. Capas Jerárquicas: Padres e Hijos

Las capas dentro de una topología pueden tener una relación jerárquica. Una capa "Padre" se puede construir a partir de los elementos de una capa "Hijo".

  • Una capa de Nivel 0 es aquella cuyos TopoElementos hacen referencia directamente a las primitivas (Nodos, Aristas, Caras).
  • Una capa de Nivel 1 sería una capa "Padre" cuyos TopoElementos hacen referencia a TopoGeometrías de una capa de Nivel 0 (su "Hijo").

Esta estructura permite construir geometrías complejas a partir de otras más simples. Por ejemplo, una capa de "Localidades" (Padre) podría construirse agrupando TopoGeometrías de una capa de "Barrios" (Hijo).

3. La Tabla de Relaciones

El "pegamento" que une todo este modelo es la tabla de relaciones. Cada topología tiene su propia tabla de relaciones (ej. topo_barrios.relation), que actúa como un puente entre las TopoGeometrías y los TopoElementos que las componen.

Cada fila en esta tabla vincula una TopoGeometría (a través de su topogeo_id y layer_id) con uno de sus componentes (a través de element_id y element_type). Una TopoGeometría compleja, como un polígono con varios huecos, tendrá múltiples entradas en esta tabla, una por cada arista o cara que la conforma.

4. Ejemplo Práctico con la Localidad de Fontibón

Continuando con el ejemplo del módulo anterior, donde creamos la topología `topo_barrios` a partir de la capa de barrios de Fontibón.

4.1. Crear una segunda capa jerárquica

Vamos a crear una nueva capa para la geometría de la Localidad de Fontibón dentro de la misma topología.

-- Añadir una columna topogeométrica a la tabla de la localidad
SELECT topology.AddTopoGeometryColumn(
    'topo_barrios',
    'public',
    'Localidad_Fontibon',
    'topogeom',
    'POLYGON'
);

4.2. Poblar la nueva capa

Ahora, poblamos la nueva columna, convirtiendo la geometría de la localidad a su representación topológica.

-- Poblar la topogeometría de la localidad
UPDATE "Localidad_Fontibon"
SET topogeom = topology.toTopoGeom(geom, 'topo_barrios', 2); -- El '2' es el nuevo layer_id

4.3. Consultar los componentes de una TopoGeometría

Ahora podemos consultar la tabla relation para ver de qué elementos primitivos se compone un barrio específico. Por ejemplo, para el barrio con `id = 10`:

-- Ver los elementos (caras, bordes, nodos) que componen el barrio con ID = 10
SELECT
    r.element_type,
    r.element_id,
    CASE
        WHEN r.element_type = 1 THEN 'Nodo'
        WHEN r.element_type = 2 THEN 'Arista'
        WHEN r.element_type = 3 THEN 'Cara'
    END AS tipo_elemento
FROM topo_barrios.relation r
JOIN "Barrios_Fontibon" b ON (b.topogeom).id = r.topogeo_id
WHERE b.id = 10 AND r.layer_id = 1; -- Filtramos por el layer_id de los barrios

Esta consulta nos mostrará una lista de todas las caras y aristas que conforman el polígono de ese barrio, demostrando cómo la TopoGeometría es en realidad una colección de referencias a elementos más simples.
