Funciones y Ejercicios de Construcción de Geometría

Creando y manipulando geometrías directamente desde SQL para resolver problemas prácticos.

1. Funciones de Construcción de Geometría

En esta sección aprenderás a construir geometrías directamente desde funciones SQL en PostGIS, sin necesidad de cargar archivos externos. Estas funciones son fundamentales cuando se requiere crear datos desde cero, hacer pruebas o complementar registros existentes con coordenadas nuevas. Trabajaremos con datos de la localidad de Teusaquillo.

1.1. Crear Puntos con `ST_MakePoint`

La función ST_MakePoint(x, y) permite construir un punto. Para que esté correctamente georreferenciado, se debe aplicar ST_SetSRID(geom, srid).

Ejemplo: Insertar una nueva IPS en Teusaquillo.

INSERT INTO ips_teusaquillo (ips_id, geometria, nombre_municipio, nombre_ips, nombre_barrio) VALUES 
(1001, ST_SetSRID(ST_MakePoint(-74.066, 4.635), 4326), 'Bogotá D.C.', 'SEDE GALERÍAS SALUD VISUAL', 'La Soledad');

1.2. Crear Líneas con `ST_MakeLine`

La función ST_MakeLine se utiliza para construir líneas conectando dos o más geometrías de tipo punto.

Ejemplo: Crear una línea que conecte los centroides de dos barrios.

SELECT ST_MakeLine(b1.geometria, b2.geometria) AS linea_conexion
FROM barrios_teusaquillo b1, barrios_teusaquillo b2
WHERE b1.nombre_barrio = 'La Soledad' AND b2.nombre_barrio = 'Quinta Paredes';

1.3. Crear Polígonos con `ST_MakePolygon`

Para construir un polígono se debe partir de una línea cerrada (un `LINESTRING` donde el primer y último punto son iguales).

Ejemplo: Crear un polígono para representar una manzana.

SELECT ST_SetSRID(
    ST_MakePolygon(
        ST_GeomFromText('LINESTRING(-74.068 4.633, -74.067 4.633, -74.067 4.634, -74.068 4.634, -74.068 4.633)')
    ), 4326
) AS poligono_manzana;

1.4. Otras Funciones Útiles

  • ST_Buffer(geometry, distance): Genera una zona de influencia (un polígono) alrededor de una geometría.
  • ST_Centroid(geometry): Calcula el centro geométrico de una geometría.
  • ST_Union(geometry): Combina múltiples geometrías en una sola, disolviendo los límites internos.
  • ST_Intersection(geomA, geomB): Devuelve la geometría que resulta de la intersección de dos geometrías.

2. Ejercicios Prácticos de Construcción

Apliquemos las funciones aprendidas para resolver problemas reales con los datos de Teusaquillo.

Ejercicio 1: Identificación de zonas sin cobertura de salud (IPS)

Problema: La Alcaldía quiere saber qué barrios no tienen una IPS a menos de 300 metros para priorizar inversión.

Solución: Creamos un buffer de 300 metros alrededor de cada IPS y luego seleccionamos los barrios que no se intersectan con ninguna de estas áreas de cobertura.

WITH zonas_cobertura AS (
    SELECT ST_Union(ST_Buffer(geometria::geography, 300)::geometry) AS geom_total
    FROM ips_teusaquillo
)
SELECT b.nombre_barrio
FROM barrios_teusaquillo b, zonas_cobertura z
WHERE NOT ST_Intersects(b.geometria, z.geom_total);

Ejercicio 2: Propuesta de nueva ciclorruta

Problema: Se desea proponer una ciclorruta aprovechando vías de tipo "residencial" de baja velocidad.

Solución: Calculamos la longitud total de todas las vías residenciales disponibles para estimar el potencial de la red de ciclorrutas.

SELECT SUM(ST_Length(geometria::geography)) AS total_metros_potenciales
FROM mallavial_teusaquillo
WHERE tipo_via ILIKE '%residencial%';

Ejercicio 3: Simulación de puntos de recolección de residuos

Problema: Se necesita registrar la ubicación de 3 nuevos puntos de reciclaje en coordenadas conocidas.

Solución: Creamos una nueva tabla e insertamos los puntos manualmente usando ST_MakePoint.

CREATE TABLE puntos_reciclaje (
    id SERIAL PRIMARY KEY,
    ubicacion geometry(Point, 4326),
    descripcion TEXT
);

INSERT INTO puntos_reciclaje (ubicacion, descripcion) VALUES
(ST_SetSRID(ST_MakePoint(-74.0665, 4.6341), 4326), 'Parque Principal'),
(ST_SetSRID(ST_MakePoint(-74.0670, 4.6352), 4326), 'Frente a clínica'),
(ST_SetSRID(ST_MakePoint(-74.0658, 4.6330), 4326), 'Zona Universitaria');

Ejercicio 4: Comparación de áreas por barrio

Problema: Se requiere identificar los 5 barrios más grandes de Teusaquillo para planificación urbana.

Solución: Usamos ST_Area sobre el tipo geography para obtener el área en metros cuadrados y ordenamos los resultados.

SELECT 
    nombre_barrio,
    ST_Area(geometria::geography) AS area_m2
FROM barrios_teusaquillo
ORDER BY area_m2 DESC
LIMIT 5;
