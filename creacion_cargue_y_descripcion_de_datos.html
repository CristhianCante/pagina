Creación, Cargue y Descripción de Datos

Unificando los pasos iniciales para preparar nuestro entorno de trabajo con datos de Bogotá.

1. Creación de una Base de Datos Espacial

El primer paso en cualquier proyecto geoespacial es configurar una base de datos que pueda manejar información geográfica. Usaremos PostgreSQL con su extensión PostGIS. La herramienta gráfica pgAdmin nos facilitará este proceso.

1.1. Creando la Base de Datos

Dentro de pgAdmin, nos conectamos a nuestro servidor y creamos una nueva base de datos. Para este ejemplo, la llamaremos CHAPINERO.

1.2. Habilitando PostGIS

Una vez creada la base de datos, debemos habilitar las capacidades espaciales. Para ello, abrimos una ventana de consulta SQL para nuestra nueva base de datos y ejecutamos el siguiente comando: 

CREATE EXTENSION postgis;

Este comando instala todas las funciones, tipos de datos y operadores espaciales que PostGIS ofrece. Para verificar que la instalación fue exitosa, podemos consultar la versión de PostGIS instalada: 

SELECT postgis_full_version();

Si todo está correcto, veremos un mensaje con los detalles de la versión de PostGIS y sus componentes (GEOS, Proj, etc.), confirmando que nuestra base de datos está lista para trabajar con datos espaciales.

2. Carga de Datos Espaciales

Con la base de datos lista, el siguiente paso es poblarla con nuestros datos geográficos. Existen varios métodos para hacerlo.

2.1. ¿Qué es un Shapefile?

El formato Shapefile es uno de los estándares más comunes para distribuir datos vectoriales. Aunque lo llamamos "un archivo", en realidad es un conjunto de archivos que trabajan juntos. Los más importantes son:

  • .shp: Almacena la geometría de las entidades (puntos, líneas o polígonos).
  • .shx: Contiene un índice que acelera el acceso a las geometrías.
  • .dbf: Una tabla en formato dBase que guarda los atributos de cada entidad.
  • .prj: (Opcional pero crucial) Define el Sistema de Referencia de Coordenadas (SRC) de los datos.

2.2. Carga con herramientas de línea de comandos

Existen herramientas potentes como shp2pgsql y ogr2ogr que permiten cargar datos desde la terminal.

Usando shp2pgsql: Esta herramienta convierte un Shapefile en sentencias SQL. El siguiente comando carga un Shapefile de barrios, le asigna el SRID 26918 y crea un índice espacial.

shp2pgsql -D -I -s 26918 barrios_chapinero.shp public.barrios_chapinero | psql -d chapinero_db -U postgres

2.3. Carga desde un archivo de respaldo (.backup)

Otra forma muy práctica es restaurar una base de datos desde un archivo de respaldo. En pgAdmin, se puede hacer clic derecho sobre la base de datos, seleccionar "Restore..." y elegir el archivo .backup. Esto cargará todas las tablas, datos y configuraciones contenidas en el respaldo.

2.4. Carga desde un cliente SIG como QGIS

También es posible conectarse a la base de datos PostGIS desde un software SIG de escritorio como QGIS. A través del "Administrador de bases de datos", se puede establecer una conexión y luego usar la función de "Importar capa/archivo" para cargar Shapefiles directamente a la base de datos de forma visual.

3. Descripción de los Datos de Chapinero

Para nuestros ejercicios, hemos cargado un conjunto de datos de la localidad de Chapinero en Bogotá. A continuación, se describen las capas y sus atributos principales.

3.1. Capa de Localidad (1 registro)

Un único polígono que representa el límite administrativo de la localidad de Chapinero.

AtributoDescripción
LocNombreNombre de la localidad (ej. 'Chapinero').
LocAreaÁrea de la localidad en unidades del SRC.
SHAPE_LengLongitud del perímetro de la geometría.
SHAPE_AreaÁrea total de la geometría.

3.2. Capa de Barrios (115 registros)

Polígonos que representan los barrios dentro de la localidad.

AtributoDescripción
barriocomuNombre del barrio o comunidad.
cod_locCódigo único de la localidad a la que pertenece.
localidadNombre de la localidad.
estadoEstado de legalización del barrio (ej. 'LEGALIZADO').

3.3. Capa de Malla Vial (3841 registros)

Líneas que representan los segmentos de la malla vial de la localidad.

AtributoDescripción
MVITIPOTipo de vía (ej. 'AK' para Avenida Carrera, 'CL' para Calle).
MVINOMBRENombre oficial o común de la vía.
MVIACALÁrea de la calzada en metros cuadrados.
Shape_LenLongitud geométrica del segmento de vía.

3.4. Capa de Incidentes (1155 registros)

Puntos que marcan la ubicación de diferentes tipos de incidentes reportados.

AtributoDescripción
FECHAFecha en que ocurrió el incidente.
ESTACIONNombre de la estación que registró el incidente.
CLASE_VEHITipo de vehículo involucrado (si aplica).
CAUSASDescripción general de la causa del incidente.
