Geometrías y Ejercicios de Geometrías

Explorando los tipos de datos espaciales y aplicando funciones de análisis en PostGIS.

1. Fundamentos de las Geometrías

Una geometría es un tipo de dato especial que representa la forma, posición y extensión de objetos geográficos. En PostGIS, estos datos se almacenan en una columna de tipo geometry, lo que nos permite realizar consultas espaciales avanzadas.

1.1. Tipos de Geometría

PostGIS soporta una gran variedad de tipos de geometría definidos por el estándar OGC (Open Geospatial Consortium). Los más comunes son:

  • POINT: Un punto individual en el espacio, definido por coordenadas (X, Y). Ideal para localizar estaciones, árboles o incidentes.
  • LINESTRING: Una secuencia de puntos conectados que forman una línea. Se usa para representar ríos, calles o rutas.
  • POLYGON: Un área cerrada definida por una o más secuencias de puntos (anillos). Representa entidades como barrios, lagos o lotes.
  • MULTI-GEOMETRÍAS (MULTIPOINT, MULTILINESTRING, MULTIPOLYGON): Colecciones de geometrías del mismo tipo. Por ejemplo, un MULTIPOLYGON para representar un archipiélago.
  • GEOMETRYCOLLECTION: Un conjunto mixto que puede contener diferentes tipos de geometrías.

1.2. Dimensiones, SRID y Campos

  • Dimensión: Cada geometría tiene una dimensión topológica. Un punto tiene dimensión 0, una línea tiene dimensión 1 y un polígono tiene dimensión 2. La función ST_Dimension() nos permite conocerla.
  • SRID (Spatial Reference System Identifier): Es un código numérico que identifica el sistema de coordenadas de la geometría. Es obligatorio definirlo correctamente para que las operaciones espaciales (como medir distancias) tengan sentido.
  • Campo Geométrico: Dentro de una tabla, la columna que almacena la información espacial se conoce como campo geométrico, comúnmente llamado geom o geometria.

2. Flujo de Trabajo: de QGIS a PostGIS

Una metodología común es utilizar un software de escritorio como QGIS para crear o preparar los datos y luego importarlos a PostGIS para un análisis más robusto y escalable a través de SQL.

2.1. Creación de Geometrías en QGIS

Puedes crear geometrías de puntos fácilmente importando un archivo de texto delimitado (CSV) que contenga columnas de coordenadas (latitud y longitud).

Una vez cargadas las capas en QGIS, puedes exportarlas directamente a tu base de datos PostGIS, asegurándote de definir el nombre de la tabla, el campo geométrico y el SRID correcto.

2.2. Creación de Geometrías directamente en PostGIS

También es posible crear la columna de geometría y poblarla directamente en PostGIS si ya tienes una tabla con coordenadas.

Paso 1: Crear una tabla para importar los datos CSV (sin geometría aún).

CREATE TABLE huecos_fontibon (
    objectid INTEGER,
    id BIGINT,
    elemento DOUBLE PRECISION,
    civ TEXT,
    nom_localidad TEXT,
    -- ... resto de columnas de atributos
    coord_x DOUBLE PRECISION,
    coord_y DOUBLE PRECISION
);

Después de crear la tabla, puedes usar la herramienta de "Import/Export Data" en pgAdmin para cargar los datos desde tu archivo CSV.

Paso 2: Añadir una columna de geometría y poblarla.

-- Añadir la columna de geometría vacía
ALTER TABLE huecos_fontibon
ADD COLUMN geom geometry(Point, 4326);

-- Poblar la nueva columna usando las coordenadas existentes
UPDATE huecos_fontibon
SET geom = ST_SetSRID(ST_MakePoint(coord_x, coord_y), 4326)
WHERE coord_x IS NOT NULL AND coord_y IS NOT NULL;

Esta operación crea un punto para cada fila de la tabla usando las columnas coord_x y coord_y, y le asigna el SRID 4326 (WGS 84), que es el estándar para coordenadas geográficas.

3. Ejercicios con Funciones Espaciales

Ahora que tenemos datos espaciales en nuestra base de datos, podemos empezar a analizarlos con funciones de PostGIS. Usaremos datos de la localidad de Fontibón como ejemplo.

3.1. Inspección de Geometrías

La función ST_AsText(geom) convierte una geometría a su formato WKT (Well-Known Text), lo que nos permite ver sus coordenadas de forma legible. 

SELECT objectid, nom_localidad, ST_AsText(geom)
FROM huecos_fontibon
LIMIT 1000;

Para obtener una lista de todas las tablas con geometrías en nuestra base de datos, podemos consultar la vista geometry_columns.

SELECT * FROM geometry_columns;

3.2. Propiedades de las Geometrías

Podemos extraer propiedades de nuestras geometrías, como su dimensión, número de vértices o coordenadas individuales.

SELECT
    ST_Dimension(geom) AS dimension,      -- Devuelve 0 para puntos
    ST_GeometryType(geom) AS tipo_geom,   -- Devuelve 'ST_Point'
    ST_NPoints(geom) AS num_vertices,     -- Devuelve 1 para puntos simples
    ST_X(geom) AS coordenada_x,           -- Extrae la coordenada X (longitud)
    ST_Y(geom) AS coordenada_y            -- Extrae la coordenada Y (latitud)
FROM huecos_fontibon
WHERE nom_localidad ILIKE 'FONTIBON'
LIMIT 1;

3.3. Consultas y Visualización

Podemos realizar consultas de atributos y visualizar el resultado geométrico directamente en el "Geometry Viewer" de pgAdmin.

-- Seleccionar todos los huecos en la localidad de Fontibón
SELECT *
FROM huecos_fontibon
WHERE nom_localidad ILIKE 'FONTIBON';

También podemos realizar consultas de agregación, como obtener una lista única de todas las localidades presentes en nuestros datos.

SELECT DISTINCT nom_localidad
FROM huecos_fontibon
ORDER BY nom_localidad;

Este tipo de ejercicios son fundamentales para limpiar, validar y entender nuestros datos antes de pasar a análisis más complejos como los joins espaciales.
