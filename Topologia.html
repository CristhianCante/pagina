Topología en Bases de Datos Espaciales

Garantizando la integridad y calidad de los datos a través de relaciones espaciales lógicas.

1. ¿Qué es la Topología?

La topología, en el contexto de los SIG, es la rama de las matemáticas que estudia las propiedades de los objetos espaciales que no cambian bajo transformaciones continuas (como estirar o doblar, pero sin romper ni pegar). Se enfoca en las relaciones de adyacencia, conectividad y contención entre geometrías.

En PostGIS, la extensión postgis_topology nos permite crear y gestionar modelos topológicos. En lugar de trabajar con geometrías simples (como en `geometry`), trabajamos con Topogeometrías, que están compuestas por elementos topológicos básicos:

  • Nodos (Nodes): Puntos que representan los extremos de las aristas.
  • Aristas (Edges): Líneas que conectan los nodos y definen los límites de las caras.
  • Caras (Faces): Polígonos que representan las áreas cerradas.

1.1. ¿Para qué sirve la Topología?

  • Garantizar la integridad de los datos: Permite detectar y corregir errores de digitalización como superposiciones, huecos (gaps) o nodos colgantes (dangles).
  • Realizar análisis de red: Es fundamental para modelar redes de transporte, hidrológicas o de servicios públicos.
  • Simplificar la edición: Al modificar un borde compartido (una arista), todas las caras adyacentes se actualizan automáticamente, manteniendo la consistencia.

2. Reglas Topológicas Comunes

Para asegurar la calidad de un modelo topológico, se deben cumplir ciertas reglas. A continuación, se presentan algunas de las más importantes para líneas y polígonos.

ReglaDescripción
No debe superponerse (Polígonos)Los polígonos de una misma capa no deben solaparse entre sí.
No debe haber huecos (Polígonos)No debe existir espacio vacío entre polígonos adyacentes en una misma capa.
No debe auto-intersectarse (Líneas)Una línea no debe cruzarse o superponerse a sí misma.
No debe intersectarse (Líneas)Las líneas de una misma capa no deben cruzarse entre sí, excepto en los nodos.

3. Funciones de Topología en PostGIS

Para trabajar con topología, primero debemos activar la extensión correspondiente.

CREATE EXTENSION postgis_topology;

PostGIS ofrece funciones para crear, validar y analizar modelos topológicos.

  • Creación: CreateTopology(), ST_CreateTopoGeo().
  • Validación: ST_IsValid(), ST_IsSimple(), ST_IsClosed().
  • Relaciones: Las mismas funciones booleanas que para geometrías (ST_Intersects, ST_Within, etc.), pero aplicadas en un contexto topológico.

4. Ejercicios Prácticos con Datos de Tunjuelito

Usaremos datos de la localidad de Tunjuelito para demostrar cómo las funciones espaciales nos ayudan a entender las relaciones topológicas.

4.1. `ST_Within` vs. `ST_Intersects`

A menudo, queremos saber qué barrios pertenecen a una localidad. Si usamos ST_Within, que es muy estricta, solo obtendremos los barrios que están completamente dentro del polígono de la localidad, excluyendo aquellos que comparten un límite.

-- Esta consulta puede devolver menos barrios de los esperados
SELECT b."SCANOMBRE", b."SCACODIGO"
FROM "BARRIOS TUNJUELITO" b, "TUNJUELITO" t
WHERE ST_Within(b.geom, t.geom);

Una función más adecuada para este caso es ST_Intersects, que incluirá todos los barrios que tocan o están dentro de la localidad, dándonos un resultado más acorde a la realidad administrativa.

-- Esta consulta es más apropiada para obtener todos los barrios de la localidad
SELECT b."SCANOMBRE", b."SCACODIGO"
FROM "BARRIOS TUNJUELITO" b, "TUNJUELITO" t
WHERE ST_Intersects(b.geom, t.geom);

4.2. Validación de Superposiciones y Validez

Podemos verificar si hay errores de digitalización, como barrios que se superponen entre sí.

-- Buscar barrios que se solapen
SELECT a."SCANOMBRE" AS barrio_a, b."SCANOMBRE" AS barrio_b
FROM "BARRIOS TUNJUELITO" a, "BARRIOS TUNJUELITO" b
WHERE a."SCACODIGO" < b."SCACODIGO" AND ST_Overlaps(a.geom, b.geom);

Si esta consulta no devuelve resultados, es una buena señal de la calidad de los datos. También podemos verificar la validez individual de cada polígono.

-- Verificar que todos los barrios sean geometrías válidas
SELECT "SCANOMBRE", ST_IsValid(geom)
FROM "BARRIOS TUNJUELITO"
WHERE NOT ST_IsValid(geom); -- Esta consulta debería devolver 0 filas

4.3. Análisis Espacial Agrupado

Finalmente, podemos usar estas relaciones para realizar análisis más complejos, como contar el número de incidentes por barrio y ordenar los resultados para identificar las zonas con mayor concentración.

-- Contar el número de incidentes por barrio y ordenar de mayor a menor
SELECT 
    b."SCANOMBRE",
    COUNT(i.id) AS total_de_incidentes
FROM "BARRIOS TUNJUELITO" b
LEFT JOIN "INCIDENTES" i ON ST_Within(i.geom, b.geom)
GROUP BY b."SCANOMBRE"
ORDER BY total_de_incidentes DESC;

Este tipo de consulta, que combina un join espacial con funciones de agregación de SQL, es la base de muchos análisis territoriales y demuestra el poder de tener datos topológicamente consistentes.
